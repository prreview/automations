## Company research notes
Hellotext sells to eCommerce and retail teams that want more revenue from customer conversations without hiring a bigger support or sales team. The main product surface is AI playbooks: cart saver, browse recovery, smart recommender, complete-the-look, price-drop alerts, first-purchase flows, and post-purchase cross-sell.

The stack from the job page is Ruby on Rails, Hotwire, Stimulus, Tailwind CSS, PostgreSQL, ElasticSearch, GitHub, JavaScript, real-time messaging, and a public JavaScript client. The GitHub repo `hellotext/hellotext.js` was pushed on 2026-06-11, which backs the active engineering signal.

The match for Raj is good on product/frontend systems, decent on backend, and weaker on Rails-specific depth. WaveMaker maps well to the frontend role because Raj works on code generation into production Next.js/React, live app architecture, custom React components, routing, and build-time systems. Rava maps to the playbook side because Raj built LLM workflows, RAG-based content features, caching, CI/CD, billing-adjacent product work, and backend pieces in a small team. Trevyn adds the AI tooling angle: repo-aware PR review with code context.

Main risk: Hellotext asks for Rails/Hotwire strength, while Raj's core stack is React, Next.js, Node, and Go. The best outreach angle is product engineering plus AI workflow ownership, then position Rails as learnable rather than pretending deep Rails background.
